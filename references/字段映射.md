## 话术节点映射

### 主流程节点填充规则

#### 1. 开场白
```
原模板: "您好，这边是{品牌}京东自营店的福利官~"
填充: {品牌} → 业务信息中的"品牌/品类"前缀
```

#### 2. 用户忙节点
```
原模板: "{活动名称}我们{品牌}给您准备了老客专属优惠..."
填充:
- {活动名称} → 活动名称关键词（如"春茶热卖季"）
- {品牌} → 品牌名
- 核心利益点 → 第一个利益点
```

#### 3. 介绍活动节点
```
原模板: "这次{活动名称}我们给您留了一份老客专属优惠！{产品列表}，{详细利益点}"
填充:
- {活动名称} → 活动名称
- {产品列表} → 主要活动产品，用逗号连接
- {详细利益点} → 所有利益点，格式化为口语化表达
```

#### 4. 挽回-起始节点
```
原模板: "稍后我先把活动通过短信发您，{次要利益点}..."
填充:
- {次要利益点} → 会员专属福利或最后一个利益点
- 如有"新人入会"相关利益点，优先使用
```

#### 5. 结束-起始节点
```
原模板: "您直接在京东搜【{品牌搜索词}】到【{店铺全称}】看看哦，{利益点总结}"
填充:
- {品牌搜索词} → 品牌名（如"竹叶青"）
- {店铺全称} → 短信签名（如"竹叶青京东自营旗舰店"）
- {利益点总结} → 核心利益点精简版
```

## 知识库生成规则

根据业务信息自动生成以下关键词问答：

| 关键词类型 | 问答内容来源 |
|------------|--------------|
| 产品类 | 主要活动产品 + 品牌/品类 |
| 价格/优惠类 | 利益点列表 |
| 时间类 | 活动时间 |
| 购买方式 | "京东搜{品牌}" |
| 会员类 | 会员相关利益点（如有） |
| 送礼类 | 茶礼/礼盒相关产品 |

## 短信模板填充规则

### 短信结构
```
【{短信签名}】{短信内容}。{利益点}。点击查看：{链接}
```

### 填充字段
- 短信签名 → 业务信息中的"短信签名"
- 短信内容 → 活动名称 + 产品关键词
- 利益点 → 核心利益点（最多3个）
- 链接 → 业务信息中的"链接"

---

## 利益点口语化处理

将业务信息中的利益点列表转换为口语化表达：

| 原始格式 | 口语化格式 |
|----------|------------|
| 单件低至6.5折 | 单件低至六五折 |
| PLUS满200减25元 | PLUS会员满两百减二十五元 |
| 爆款茶礼8.8折起 | 爆款茶礼八八折起 |
| 新人入会享优惠券 | 新人入会还能领优惠券 |

规则：
- 数字转换为中文口语表达（可选）
- "元"字可省略
- "享"改为"领"或"拿"更口语化
