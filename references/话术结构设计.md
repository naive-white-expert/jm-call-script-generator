# 外呼话术结构设计

## 话术三层结构

话术分为三个层次：**流程 → 节点 → 答案（播报内容）**

```
流程1
  ├── 节点A
  │     ├── 播报内容
  │     ├── 用户回复选项1 → 节点B
  │     ├── 用户回复选项2 → 节点C
  │     └── 默认条件 → 节点D
  ├── 节点B
  │     └── ...
  └── 节点C
        └── ...

流程2
  ├── 节点E
  │     └── ...
  └── ...
```

---

## 流程执行规则

### 流程间执行顺序

流程按**数字标号从小到大**顺序执行：

```
1-开始流程 → 2-介绍活动 → 3-挽回 → 4-结束
```

**流程命名规范：**
- 格式：`{数字}-{流程名称}`
- 示例：`1-开始流程`、`2-介绍活动`、`3-挽回`、`4-结束`

**流程跳转：**
- 当前流程完成后，通过"流程跳转"节点跳转到下一个流程
- 节点名称：`流程跳转 - {目标流程名称}`
- 示例：`流程跳转 - 2-介绍活动`

## 节点执行规则

### 节点间跳转逻辑

节点之间按**用户回复**选择下一个节点：

```
当前节点
  ├── 用户回复: "未听清重述" → 下一步: 当前节点（循环）
  ├── 用户回复: "客户忙"    → 下一步: 用户忙节点
  ├── 用户回复: "拒绝"      → 下一步: 挽回/挂断节点
  ├── 用户回复: "肯定"      → 下一步: 邀约成功/跳转节点
  └── 用户回复: "默认条件"  → 下一步: 默认跳转节点
```

### 用户回复类型

| 回复类型 | 说明 | 跳转逻辑 |
|----------|------|----------|
| 未听清重述 | 用户没听清，需要重说 | → 当前节点（重新播报） |
| 客户忙 | 用户表示现在忙 | → 用户忙节点 |
| 拒绝 | 用户明确拒绝 | → 挽回流程 / 挂断 |
| 肯定 | 用户同意/感兴趣 | → 下一流程 / 邀约成功 |
| 默认条件 | 无明确回复时的默认处理 | → 根据流程设计跳转 |

### 节点命名规范

| 节点类型 | 命名示例 | 说明 |
|----------|----------|------|
| 起始节点 | `开场白`、`介绍活动` | 流程的第一个播报节点 |
| 状态节点 | `用户忙`、`客户拒绝` | 根据用户回复进入的状态 |
| 跳转节点 | `跳转流程节点`、`流程跳转 - xxx` | 用于流程间跳转 |
| 结束节点 | `挂断` | 结束通话 |

## 播报内容设计

### 播报内容结构

播报内容 = **开场钩子 + 核心信息 + 引导行动**

```
示例：
"这次春茶热卖季我们给您留了一份老客专属优惠！（钩子）
 竹叶青绿茶、茉莉花茶都有，单件低至6.5折。（核心信息）
 邀您来竹叶青京东自营旗舰店看看呢~（引导行动）"
```

### 播报内容长度建议

| 节点类型 | 字数建议 | 说明 |
|----------|----------|------|
| 开场白 | 20-30字 | 简短问候，确认身份 |
| 用户忙 | 40-60字 | 快速抛出钩子，争取继续 |
| 介绍活动 | 80-120字 | 详细介绍产品+利益点 |
| 挽回 | 60-80字 | 强调次要利益点，争取转化 |
| 结束 | 50-70字 | 总结+引导搜索+祝福 |

## 流程设计模板

### 标准4流程结构

```
1-开始流程（开场阶段）
  ├── 开场白（确认身份）
  ├── 用户忙（争取继续）
  ├── 用户依然忙 → 挂断
  ├── 忙拒绝 → 挂断
  └── 跳转流程节点 → 2-介绍活动

2-介绍活动（核心介绍）
  ├── 介绍活动（详细介绍产品+利益点）
  ├── 短信-用户忙 → 挂断
  ├── 客户拒绝 → 3-挽回
  └── 邀约成功 → 4-结束

3-挽回（转化挽回）
  ├── 挽回-起始节点（强调次要利益点）
  ├── 挽回成功 → 挂断
  └── 挽回失败 → 挂断

4-结束（收尾）
  ├── 结束-起始节点（总结+引导）
  ├── 短信-结束 → 挂断
  └── 最终拒绝 → 挂断
```

## Excel 表格结构

### 主流程 Sheet 列定义

| 列名 | 说明 | 示例 |
|------|------|------|
| 流程名称 | 流程标识 | `1-开始流程` |
| 节点名称 | 当前节点名 | `开场白` |
| 播报条件 | 触发条件（可选） | 空 |
| 播报内容 | AI播报的话术文本 | `您好，这边是xxx...` |
| 用户回复 | 用户可能的回复类型 | `未听清重述` |
| 下一步执行节点 | 跳转目标节点 | `开场白` 或 `用户忙` |
| 意向标签 | 挂断时打的标签 | `用户忙`、`同意邀约` |
| 是否触发短信 | 是否发短信 | 空 |
| 短信模板名称 | 对应短信模板 | 空 |

### 行合并规则

同一节点的多个用户回复选项，**流程名称和节点名称列可合并**：

```
| 1-开始流程 | 开场白 |  | 播报内容... | 未听清重述 | 开场白 |  |  |  |
|           |        |  |            | 客户忙    | 用户忙 |  |  |  |
|           |        |  |            | 默认条件  | 跳转流程节点 |  |  |  |
```

## 设计要点

1. **流程编号必须有序**：从1开始递增，确保执行顺序正确
2. **每个流程必须有入口节点**：流程的第一个节点作为入口
3. **节点跳转要闭环**：所有路径最终都要指向挂断或下一流程
4. **避免死循环**：未听清重述最多循环3次，应设置超时挂断
5. **意向标签要准确**：挂断节点必须打上正确的意向标签
