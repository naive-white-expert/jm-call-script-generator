根据业务活动信息和模板文档，自动生成完整的AI外呼话术模板。

## 一、输出结构约束

### 1.1 三级结构不变原则

**核心原则**：只替换内容，结构骨架完全保留

```
原模板结构                    填充后结构
─────────────────────────────────────────────
流程1                         流程1（名称不变）
  ├── 节点A（播报内容含占位符）    ├── 节点A（播报内容已替换）
  │     ├── 用户回复选项1         │     ├── 用户回复选项1（不变）
  │     ├── 用户回复选项2         │     ├── 用户回复选项2（不变）
  │     └── 默认条件              │     └── 默认条件（不变）
  ├── 节点B                       ├── 节点B（不变）
  └── ...                         └── ...
流程2                         流程2（名称不变）
  └── ...                         └── ...
```

### 1.2 约束规则

| 约束项 | 规则 | 说明 |
|--------|------|------|
| 流程名称 | ❌ 不修改 | 保持原模板的流程编号和名称 |
| 流程顺序 | ❌ 不修改 | 保持原模板的流程执行顺序 |
| 节点名称 | ❌ 不修改 | 保持原模板的节点名称 |
| 节点跳转 | ❌ 不修改 | 保持原模板的用户回复 → 下一步节点映射 |
| 播报内容 | ✅ 替换占位符 | 只替换品牌、活动、产品、利益点等业务信息 |
| 意向标签 | ❌ 不修改 | 保持原模板的挂断标签 |
| 短信触发 | ❌ 不修改 | 保持原模板的短信触发逻辑 |

## 二、模板分析

### 2.1 目标

在填充话术前，先分析模板，识别三类内容：

| 内容类型 | 说明 | 处理方式 |
|----------|------|----------|
| **可替换内容** | 业务相关信息（品牌、活动、产品、利益点等） | 根据业务信息替换 |
| **行业特定内容** | 原模板所属行业的专有词汇 | 替换为新行业对应词汇 |
| **不可替换内容** | 语言风格、表达方式、语气词、句式结构 | 完整保留 |

### 2.2 可替换内容识别

**识别方法**：扫描模板播报内容，识别以下模式：

| 模式类型 | 示例 | 说明 |
|----------|------|------|
| 显式占位符 | `{品牌}`、`{活动名称}` | 明确标记的替换点 |
| 隐式品牌名 | 原模板中的具体品牌名（如"娇韵诗"） | 需替换为新品牌 |
| 隐式活动名 | 原模板中的具体活动名 | 替换为新活动 |
| 隐式产品名 | 原模板中的具体产品名 | 需替换为新产品 |
| 隐式利益点 | 原模板中的具体优惠内容 | 替换为新利益点 |

### 2.3 行业特定内容识别

**识别原模板行业词汇**：

| 原行业 | 示例词汇 | 替换目标 |
|--------|----------|----------|
| 护肤品 | 精华、面霜、眼霜、护肤品、正装 | 茶叶相关词汇 |
| 食品 | 零食、饮料、保质期 | 对应行业词汇 |
| 服装 | 款式、尺码、面料 | 对应行业词汇 |

**替换原则**：
- 识别原模板所属行业
- 确定目标行业
- 建立行业词汇映射表
- 批量替换行业特定词汇

### 2.4 语言风格提取

**提取维度**：

| 维度 | 说明 | 示例 |
|------|------|------|
| 语气词 | 句首/句尾的语气表达 | "您好~"、"呢~"、"哦~" |
| 句式结构 | 播报内容的句子组成 | "钩子+核心信息+引导行动" |
| 称呼方式 | 对用户的称呼 | "您"（尊称）、"亲"（亲昵） |
| 表达风格 | 正式/口语/亲切 | "这边是..." vs "我是..." |
| 结尾符号 | 句尾符号风格 | "~"表示友好，"！"表示强调 |

## 三、信息提取流程（v3.0 优化版）

### 3.1 流程概览

```
步骤1: 总结活动信息 → 步骤2: 生成话术要点 → 步骤3: 分析模板结构 → 步骤4: 内容替换填充 → 步骤5: 校验输出
    ↓                    ↓                    ↓                    ↓                    ↓
 提取活动要素        基于活动生成核心话术    识别模板可替换位置     精准替换保持格式      检查修复输出
```

**流程优化说明**：
- **旧流程问题**：直接从活动信息跳到内容替换，缺少"话术要点生成"环节，导致活动名称、利益点表达不准确
- **新流程优势**：先完整理解活动，再生成话术要点，最后匹配模板，确保内容准确自然

---

### 3.2 步骤 1：总结活动信息

#### 3.2.1 活动信息提取框架

| 大类编号 | 大类名称 | 必要性 | 说明 |
|----------|----------|--------|------|
| C1 | 品牌信息 | ✅ 必填 | 品牌名称、店铺名称、搜索关键词 |
| C2 | 活动信息 | ✅ 必填 | 活动名称、活动时间、活动目的 |
| C3 | 产品信息 | ✅ 必填 | 主要产品、产品特点 |
| C4 | 利益点信息 | ✅ 必填 | 优惠内容、会员福利、参与门槛 |
| C5 | 渠道信息 | ⚠️ 选填 | 短信签名、链接、店铺地址 |
| C6 | 行业信息 | ✅ 必填 | 所属行业 |

#### 3.2.2 ⚠️ 活动名称提取规则（重要）

**常见错误**：
| 错误类型 | 错误示例 | 正确理解 |
|----------|----------|----------|
| 把权益内容当活动名 | "会员升级权益" | 这是权益内容，不是活动名称 |
| 太直接太生硬 | "老客户唤醒" | 需要包装成委婉正式的表达 |

**活动名称提取方法**：

**方法1：从活动主题字段直接提取**
```
原文：活动主题：阅读狂欢节
提取：活动名称 = "阅读狂欢节"
```

**方法2：从触达需求推断（无明确活动主题时）**

| 触达需求关键词 | 活动性质 | 包装后活动名称 |
|----------------|----------|----------------|
| "沉睡客"+"引导/领取" | 老客户唤醒 | **老会员权益续享** 或 **高卡级一键回归** |
| "新客"+"邀请/拉新" | 新客拉新 | **新客专享礼遇** |
| "老客"+"回馈/福利" | 老客回馈 | **老客户感恩回馈** |
| "会员"+"升级/权益" | 会员升级 | **会员升级专享礼** |

**示例**：
```
原文：触达需求 = "引导沉睡客领取专属升级权益"
分析：
  - "沉睡客" → 老客户/老会员
  - "引导...领取" → 邀请续享
  - "专属升级权益" → 权益内容（非活动名）
推断：活动名称 = "老会员权益续享" 或 "高卡级一键回归"
```

**方法3：从活动目的推断**
```
原文：活动目的 = "活动期间按消费金额，让曾经的高等级会员恢复对应卡级"
关键词：高等级会员 + 恢复卡级
推断：活动名称 = "高卡级一键回归"
```

#### 3.2.3 利益点信息提取规则

**提取要点**：
| 要素 | 说明 | 示例 |
|------|------|------|
| 参与门槛 | 满多少元/什么等级才能参与 | "实付满2000元" |
| 核心权益 | 主要获得的福利 | "恢复银卡等级"、"时光琉璃家族2件套" |
| 档位区分 | 不同等级对应不同权益 | 银卡档/金卡档 |
| 参与方式 | 如何参与/领取 | "确认收货后30天内联系客服登记" |
| 限制条件 | 次数限制/时间限制 | "每人限1次"、"活动期内有效" |

**示例**：
```
原文档：
  银卡回归：实付满 2000 元 → 恢复银卡
  金卡回归：实付满 4000 元 → 恢复金卡（含 SPA 奢护 60 分钟体验卡）

提取为：
{
  "银卡档": {
    "门槛": "实付满2000元",
    "权益": "恢复银卡等级"
  },
  "金卡档": {
    "门槛": "实付满4000元", 
    "权益": "恢复金卡等级 + SPA奢护60分钟体验卡"
  },
  "参与方式": "确认收货后30个工作日内联系客服登记办理"
}
```

### 3.3 步骤 2：生成话术要点

**目的**：基于活动信息，先生成关键话术片段，再填入模板

#### 3.3.1 必须生成的话术要点

| 要点类型 | 生成内容 | 示例 |
|----------|----------|------|
| 活动介绍 | 一句话概括活动核心 | "银卡及以上老会员实付满2000元就能恢复银卡等级" |
| 核心利益点 | 最吸引人的权益表达 | "实付满4000元还能恢复金卡等级并享SPA奢护体验" |
| 参与引导 | 如何参与的口语化表达 | "确认收货后30天内联系客服登记就行" |
| 时间紧迫 | 活动结束时间 | "活动到4月29日就结束了" |

#### 3.3.2 话术要点示例

**活动信息**：
```
活动名称：高卡级一键回归
活动时间：4月6日—4月29日
银卡档：实付满2000元恢复银卡
金卡档：实付满4000元恢复金卡（含SPA体验）
参与方式：确认收货后30天内联系客服登记
```

**生成的话术要点**：
```
【开场利益点】
"银卡及以上老会员实付满2000元就能恢复银卡等级"

【核心介绍】
"「高卡级一键回归」活动，银卡及以上老会员实付满2000元就能恢复银卡等级，满4000元还能恢复金卡等级并享SPA奢护体验"

【参与引导】
"确认收货后30天内联系客服登记就行"

【时间紧迫】
"活动到4月29日就结束了"
```

### 3.4 步骤 3：分析模板结构

**目标**：识别模板中哪些内容需要替换，哪些需要保留

#### 3.4.1 模板内容分类

| 内容类型 | 说明 | 处理方式 |
|----------|------|----------|
| **可替换内容** | 品牌、活动、产品、利益点等业务信息 | 用话术要点替换 |
| **行业特定内容** | 原模板行业的专有词汇 | 替换为新行业对应词汇 |
| **不可替换内容** | 语言风格、语气词、句式结构 | 完整保留 |

#### 3.4.2 格式保持要求

| 格式项 | 保持方法 |
|--------|----------|
| 合并单元格 | 复制原模板，只修改单元格值，不删除/新增合并区域 |
| 列数/行数 | 严格不变 |
| 列宽/行高 | 复制原模板时自动保持 |

### 3.5 步骤 4：内容替换填充

#### 3.5.1 替换优先级

```
优先级从高到低：
1. 长词组优先（"法国娇韵诗京东自营旗舰店" > "娇韵诗"）
2. 完整词组优先（"京东自营旗舰店" > "京东自营"）
3. 行业词汇最后替换
```

#### 3.5.2 ⚠️ 无对应内容删除原则

**核心规则**：原模板中的内容，如果在新活动信息中没有对应内容，必须删除！

| 原模板内容 | 新信息状态 | 处理 |
|------------|------------|------|
| "38节活动" | 新活动名为"高卡级一键回归" | 替换为新活动名 |
| "叠加美妆券" | 新活动无此内容 | 删除 |
| "满300减50" | 新利益点列表中无此项 | 删除 |

#### 3.5.3 口语化处理

| 原格式 | 口语化格式 |
|--------|------------|
| 6.5折 | 六五折 |
| 200减25 | 两百减二十五 |
| 享优惠券 | 领券 |

## 四、自动校验机制

| 规则编号 | 规则名称 | 检查内容 | 严重程度 |
|----------|----------|----------|----------|
| R1 | 列数一致性 | 输出文件列数必须与模板一致 | 🔴 高 |
| R2 | 知识库条目减少 | 知识库条目数量不能少于原模板 | 🔴 高 |
| R3 | 店铺名重复 | "旗舰店旗舰店"、"自营店自营" | 🔴 高 |
| R4 | 行业词汇残留 | 原行业词汇未替换（护肤品、精华等） | 🔴 高 |
| R5 | 品牌名残留 | 原品牌名未替换（娇韵诗） | 🔴 高 |
| R6 | 活动名残留 | 原活动名未替换（38节） | 🔴 高 |
| R7 | 空播报内容 | 播报内容为空或只有空格 | 🔴 高 |
| R8 | 短信内容重复 | "PLUS会员PLUS"等重复表达 | 🔴 高 |
| R9 | 链接缺失 | 短信模板缺少链接 | 🟡 中 |
| R10 | 语气词缺失 | 关键节点缺少语气词 | 🟢 低 |
| R11 | 无对应内容残留 | 原模板中的活动/品牌/利益点在新信息中无对应但仍保留 | 🔴 高 |
| R12 | 数量词搭配不当 | "一张权益"、"一个活动"等不通顺表达 | 🔴 高 |

### 4.2 校验流程

```
生成话术 → 执行校验规则 → 收集问题 → 自动修复 → 再次校验 → 输出结果
                              ↓
                        无法自动修复 → 标记待人工确认
```

### 4.3 自动修复逻辑

| 问题类型 | 自动修复方式 |
|----------|--------------|
| 店铺名重复 | 去重（"旗舰店旗舰店" → "旗舰店"） |
| 行业词汇残留 | 按映射表替换 |
| 品牌名残留 | 替换为新品牌 |
| 活动名残留 | 替换为新活动 |
| 口语化重复 | 去重 |
| 无对应内容残留 | 删除原模板中无新信息对应的内容 |
| 数量词搭配不当 | 按搭配表修正（"一张权益"→"一份权益"） |

### 4.4 数量词搭配规范

**常见搭配错误及修正**：

| ❌ 错误表达 | ✅ 正确表达 | 说明 |
|-------------|-------------|------|
| 一张权益 | 一份权益 | 权益用"份"更通顺 |
| 一个活动 | 一场活动 / 这次活动 | 活动用"场"或省略量词 |
| 一个优惠 | 一项优惠 / 这个优惠 | 优惠用"项"更正式 |
| 一张优惠券 | 一张优惠券 | 正确，保留 |
| 一个福利 | 一项福利 / 这个福利 | 福利用"项"更正式 |
| 一个礼品 | 一份礼品 | 礼品用"份" |
| 一个产品 | 一款产品 | 产品用"款" |
| 一个权益 | 一项权益 | 权益用"项"或"份" |

**量词搭配规则**：

| 名词类型 | 推荐量词 | 示例 |
|----------|----------|------|
| 权益、福利 | 份、项 | 一份权益、一项福利 |
| 活动 | 场、次 | 一场活动、这次活动 |
| 优惠、折扣 | 项、个 | 一项优惠、这个折扣 |
| 产品、商品 | 款、个 | 一款产品、这个商品 |

### 7.3 核心原则

1. **活动优先原则**：先完整理解活动，再生成话术要点，最后匹配模板
2. **结构不变原则**：输出文件必须严格保持原模板的列结构，只修改内容
3. **知识库只增不减原则**：保持原有知识库条目，只修改内容或新增
4. **校验优先原则**：生成后必须通过所有校验规则
5. **格式完全保持原则**：合并单元格、列宽等格式必须完全保持

### 7.4 Excel 格式保持规范

**格式保持是关键要求，必须严格遵循！**

#### 7.4.1 必须保持的格式项

| 格式项 | 说明 | 保持方法 |
|--------|------|----------|
| **合并单元格** | 原模板所有合并区域 | 复制原模板文件，直接修改单元格值，不删除/新增合并区域 |
| **列宽** | 每列的宽度设置 | 复制原模板时自动保持 |
| **行高** | 每行的高度设置 | 复制原模板时自动保持 |
| **列数** | 每个Sheet的列数 | 严格不变，不能增减列 |
| **行数** | 每个Sheet的行数 | 严格不变，不能增减行 |
| **Sheet顺序** | Sheet的排列顺序 | 保持原顺序 |

#### 7.4.2 正确的生成流程

```
加载原模板（load_workbook）
    ↓
遍历所有单元格，只修改文本内容
    ↓
注意：合并单元格只修改左上角单元格的值
    ↓
保存新文件（save）
    ↓
验证：合并单元格数量、列数、行数必须与原模板一致
```

#### 7.4.3 合并单元格处理规则

**关键点：合并单元格区域内，只有左上角单元格有值，其他单元格为空**

```python
# 正确做法：检查单元格是否在合并区域内
for merged_range in ws.merged_cells.ranges:
    if cell.coordinate in merged_range:
        # 只修改左上角单元格
        if cell.coordinate == merged_range.start_cell.coordinate:
            cell.value = replace_text(cell.value)
        # 其他单元格跳过（它们没有独立值）
        break
```

#### 7.4.4 格式验证清单

生成后必须验证：

```python
# 验证格式保持
for sheet_name in wb_new.sheetnames:
    ws_new = wb_new[sheet_name]
    ws_old = wb_old[sheet_name]
    
    # 合并单元格数量必须一致
    assert len(ws_new.merged_cells.ranges) == len(ws_old.merged_cells.ranges)
    
    # 列数必须一致
    assert ws_new.max_column == ws_old.max_column
    
    # 行数必须一致
    assert ws_new.max_row == ws_old.max_row
```

#### 7.4.5 已发现的格式问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 合并单元格丢失 | 使用新创建的Workbook而非复制原模板 | 使用 `load_workbook` 加载原模板，直接修改后保存 |
| 列宽变化 | 创建新Sheet时未设置列宽 | 复制原模板，列宽自动保持 |
| 行数增加 | 添加了新行 | 只修改现有单元格，不新增行 |
