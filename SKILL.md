# JM外呼-话术生成器

根据业务活动信息和模板文档，自动生成完整的AI外呼话术模板。

---

## 一、输出结构约束

### 1.1 三级结构不变原则

**核心原则**：只替换内容，结构骨架完全保留

```
原模板结构                    填充后结构
─────────────────────────────────────────────
流程1                         流程1（名称不变）
  ├── 节点A（播报内容含占位符）    ├── 节点A（播报内容已替换）
  │     ├── 用户回复选项1         │     ├── 用户回复选项1（不变）
  │     ├── 用户回复选项2         │     ├── 用户回复选项2（不变）
  │     └── 默认条件              │     └── 默认条件（不变）
  ├── 节点B                       ├── 节点B（不变）
  └── ...                         └── ...
流程2                         流程2（名称不变）
  └── ...                         └── ...
```

### 1.2 约束规则

| 约束项 | 规则 | 说明 |
|--------|------|------|
| 流程名称 | ❌ 不修改 | 保持原模板的流程编号和名称 |
| 流程顺序 | ❌ 不修改 | 保持原模板的流程执行顺序 |
| 节点名称 | ❌ 不修改 | 保持原模板的节点名称 |
| 节点跳转 | ❌ 不修改 | 保持原模板的用户回复 → 下一步节点映射 |
| 播报内容 | ✅ 替换占位符 | 只替换品牌、活动、产品、利益点等业务信息 |
| 意向标签 | ❌ 不修改 | 保持原模板的挂断标签 |
| 短信触发 | ❌ 不修改 | 保持原模板的短信触发逻辑 |

## 二、模板分析

### 2.1 目标

在填充话术前，先分析模板，识别三类内容：

| 内容类型 | 说明 | 处理方式 |
|----------|------|----------|
| **可替换内容** | 业务相关信息（品牌、活动、产品、利益点等） | 根据业务信息替换 |
| **行业特定内容** | 原模板所属行业的专有词汇 | 替换为新行业对应词汇 |
| **不可替换内容** | 语言风格、表达方式、语气词、句式结构 | 完整保留 |

### 2.2 可替换内容识别

**识别方法**：扫描模板播报内容，识别以下模式：

| 模式类型 | 示例 | 说明 |
|----------|------|------|
| 显式占位符 | `{品牌}`、`{活动名称}` | 明确标记的替换点 |
| 隐式品牌名 | 原模板中的具体品牌名（如"娇韵诗"） | 需替换为新品牌 |
| 隐式活动名 | 原模板中的具体活动名 | 替换为新活动 |
| 隐式产品名 | 原模板中的具体产品名 | 需替换为新产品 |
| 隐式利益点 | 原模板中的具体优惠内容 | 替换为新利益点 |

### 2.3 行业特定内容识别

**识别原模板行业词汇**：

| 原行业 | 示例词汇 | 替换目标 |
|--------|----------|----------|
| 护肤品 | 精华、面霜、眼霜、护肤品、正装 | 茶叶相关词汇 |
| 食品 | 零食、饮料、保质期 | 对应行业词汇 |
| 服装 | 款式、尺码、面料 | 对应行业词汇 |

**替换原则**：
- 识别原模板所属行业
- 确定目标行业
- 建立行业词汇映射表
- 批量替换行业特定词汇

### 2.4 语言风格提取

**提取维度**：

| 维度 | 说明 | 示例 |
|------|------|------|
| 语气词 | 句首/句尾的语气表达 | "您好~"、"呢~"、"哦~" |
| 句式结构 | 播报内容的句子组成 | "钩子+核心信息+引导行动" |
| 称呼方式 | 对用户的称呼 | "您"（尊称）、"亲"（亲昵） |
| 表达风格 | 正式/口语/亲切 | "这边是..." vs "我是..." |
| 结尾符号 | 句尾符号风格 | "~"表示友好，"！"表示强调 |

## 三、信息提取流程

### 3.1 流程概览

```
步骤A: 总结大类 → 步骤B: 内容梳理 → 步骤C: 内容替换 → 步骤D: 自动校验 → 步骤E: 生成输出
    ↓                  ↓                  ↓                  ↓                  ↓
 识别信息类型        按类型提取         匹配+行业适配       检查问题修复        输出Excel+标注
```

### 3.2 步骤 A：总结客户信息大类

**大类定义（预设）**：

| 大类编号 | 大类名称 | 必要性 | 说明 |
|----------|----------|--------|------|
| C1 | 品牌信息 | ✅ 必填 | 品牌名称、店铺名称、搜索关键词 |
| C2 | 活动信息 | ✅ 必填 | 活动名称、活动时间 |
| C3 | 产品信息 | ✅ 必填 | 主要产品、产品特点 |
| C4 | 利益点信息 | ✅ 必填 | 优惠内容、会员福利、限时信息 |
| C5 | 渠道信息 | ⚠️ 选填 | 短信签名、链接、店铺地址 |
| C6 | 行业信息 | ✅ 必填 | 所属行业、原模板行业 |

### 3.3 步骤 B：内容梳理

**输出格式**：

```json
{
  "C1_品牌信息": {
    "品牌名称": "竹叶青",
    "店铺全称": "竹叶青京东自营旗舰店",
    "品牌搜索词": "竹叶青"
  },
  "C2_活动信息": {
    "活动名称": "春茶热卖季",
    "活动时间": "3月29日 20:00 - 3月31日 23:59"
  },
  "C3_产品信息": {
    "主要产品": ["竹叶青绿茶", "碧潭飘雪茉莉花茶", "万紫千红红茶"],
    "产品特点": "明前新茶，口感鲜爽"
  },
  "C4_利益点信息": {
    "利益点列表": ["单件低至6.5折", "PLUS满200减25", "爆款茶礼8.8折起"],
    "核心利益点": "单件低至6.5折",
    "次要利益点": "新人入会领券"
  },
  "C5_渠道信息": {
    "短信签名": "竹叶青京东自营旗舰店",
    "链接": "https://shop.m.jd.com/?shopId=xxx"
  },
  "C6_行业信息": {
    "行业": "茶叶",
    "原行业": "护肤品"
  }
}
```

### 3.4 步骤 C：内容替换

**替换优先级（避免重复替换）**：

```
优先级从高到低：
1. 长词组优先（"法国娇韵诗京东自营旗舰店" > "娇韵诗"）
2. 完整词组优先（"京东自营旗舰店" > "京东自营"）
3. 行业词汇最后替换
```

**替换映射表示例**：

| 原文 | 替换目标 | 类型 |
|------|----------|------|
| 法国娇韵诗京东自营旗舰店 | {店铺全称} | 品牌+店铺 |
| 娇韵诗京东自营店 | {店铺全称} | 品牌+店铺 |
| 娇韵诗 | {品牌名称} | 品牌 |
| 38节 | {活动名称} | 活动 |
| 护肤品 | {行业} | 行业适配 |
| 精华 | {产品} | 行业适配 |

**口语化处理规则**：

| 原格式 | 口语化格式 |
|--------|------------|
| 6.5折 | 六五折 |
| 200减25 | 两百减二十五 |
| 享优惠券 | 领券 |

## 四、自动校验机制

### 4.1 校验规则清单

| 规则编号 | 规则名称 | 检查内容 | 严重程度 |
|----------|----------|----------|----------|
| R1 | 列数一致性 | 输出文件列数必须与模板一致 | 🔴 高 |
| R2 | 知识库条目减少 | 知识库条目数量不能少于原模板 | 🔴 高 |
| R3 | 店铺名重复 | "旗舰店旗舰店"、"自营店自营" | 🔴 高 |
| R4 | 行业词汇残留 | 原行业词汇未替换（护肤品、精华等） | 🔴 高 |
| R5 | 品牌名残留 | 原品牌名未替换（娇韵诗） | 🔴 高 |
| R6 | 活动名残留 | 原活动名未替换（38节） | 🔴 高 |
| R7 | 空播报内容 | 播报内容为空或只有空格 | 🔴 高 |
| R8 | 短信内容重复 | "PLUS会员PLUS"等重复表达 | 🔴 高 |
| R9 | 链接缺失 | 短信模板缺少链接 | 🟡 中 |
| R10 | 语气词缺失 | 关键节点缺少语气词 | 🟢 低 |

### 4.2 校验流程

```
生成话术 → 执行校验规则 → 收集问题 → 自动修复 → 再次校验 → 输出结果
                              ↓
                        无法自动修复 → 标记待人工确认
```

### 4.3 自动修复逻辑

| 问题类型 | 自动修复方式 |
|----------|--------------|
| 店铺名重复 | 去重（"旗舰店旗舰店" → "旗舰店"） |
| 行业词汇残留 | 按映射表替换 |
| 品牌名残留 | 替换为新品牌 |
| 活动名残留 | 替换为新活动 |
| 口语化重复 | 去重 |

### 4.4 校验结果输出

**新增 sheet：校验结果**

| 列名 | 说明 |
|------|------|
| 节点 | 问题所在节点 |
| 规则 | 触发的校验规则 |
| 问题 | 问题描述 |
| 状态 | 已修复 / 待人工确认 |
| 内容片段 | 问题内容预览 |

## 五、输出文件结构

### 5.1 Sheet 组成

| Sheet | 内容 | 说明 |
|-------|------|------|
| 主流程 | 填充后的对话流程 | 结构保留，内容已替换 |
| 知识库 | 关键词问答库 | 根据业务信息生成 |
| 短信 | 短信模板 | 已替换品牌、活动、链接 |
| 替换标注 | 替换/保留记录 | 方便运营检查 |
| 校验结果 | 问题及修复记录 | 质量保证 |

### 5.2 替换标注格式

| 列名 | 说明 | 示例 |
|------|------|------|
| 流程名称 | 所属流程 | `1-开始流程` |
| 节点名称 | 所属节点 | `开场白` |
| 原播报内容 | 模板原文 | `诶您好，这边是娇韵诗...` |

| 替换内容清单 | 被替换的内容 | `娇韵诗→竹叶青` |
| 保留内容清单 | 被保留的风格 | `诶您好、~、这边是` |
| 替换状态 | 完整/部分/异常 | `✅ 完整替换` |

## 六、输出展示规范

### 6.1 展示目标

生成话术后，必须清晰展示给运营确认：
1. **主流程**：每个节点的原内容 vs 新内容完整对比
2. **知识库**：所有关键词问答的完整内容
3. **短信**：所有短信模板的完整内容

### 6.2 Excel 输出结构

生成文件包含 3 个 Sheet：

| Sheet | 内容 | 说明 |
|-------|------|------|
| 主流程完整对比 | 原播报内容 vs 新播报内容 | 标黄显示变动部分 |
| 知识库完整内容 | 关键词 + 回复内容 | 按序号排列 |
| 短信完整内容 | 模板名称 + 内容 + 触发节点 | 完整短信文本 |

### 6.3 主流程展示格式

**表头**：
| 流程名称 | 节点名称 | 原播报内容（完整） | 新播报内容（完整） | 变动说明 |

**标黄规则**：
- 新播报内容有变动的单元格标黄
- 变动说明列标黄

**示例**：

| 流程名称 | 节点名称 | 原播报内容 | 新播报内容 | 变动说明 |
|----------|----------|------------|------------|----------|
| 1-开始流程 | 开场白 | 诶您好，这边是娇韵诗京东自营店的福利官~ | 诶您好，这边是竹叶青京东自营旗舰店的福利官~ | 🟡 内容重构 |
| 1-开始流程 | 用户忙 | 您别挂哈，38节我们娇韵诗... | 您别挂哈，春茶热卖季我们竹叶青... | 🟡 内容重构 |
| 1-开始流程 | 用户依然忙 | 那您先忙，得空可以看下... | 那您先忙，得空可以看下... | ✓ 无变动 |

### 6.4 知识库展示格式

**表头**：
| 序号 | 关键词 | 回复内容 |

**示例**：

| 序号 | 关键词 | 回复内容 |
|------|--------|----------|
| 1 | 价格 | 这次春茶热卖季，单件六五折，爆款茶礼八八折起 |
| 2 | 优惠 | PLUS会员满两百减二十五，新人入会还能领券 |
| 3 | 茶叶 | 我们有竹叶青绿茶、碧潭飘雪茉莉花茶、万紫千红红茶 |

### 6.5 短信展示格式

**表头**：
| 短信模板名称 | 短信内容 | 触发节点 |

**示例**：

| 短信模板名称 | 短信内容 | 触发节点 |
|--------------|----------|----------|
| 开场短信 | 【竹叶青京东自营旗舰店】春茶热卖季来啦！竹叶青新茶六五折... | 用户依然忙、忙拒绝 |

### 6.6 文本展示格式

在对话中向用户展示时，按以下格式：

**主流程关键节点**：
```
【节点名称】
原内容: {完整原文}

```

```

...
```


## 七、经验积累

### 7.1 已发现的问题及解决方案

| 问题 | 原因 | 解决方案 | 加入版本 |
|------|------|----------|----------|
| 列数不一致 | 生成对比表格而非实际话术 | 严格保持原模板列结构，只修改播报内容列 | v2.1 |
| 知识库条目减少 | 重新生成知识库 | 保持原有条目，只修改内容或新增 | v2.1 |
| 短信内容重复 | 口语化处理叠加 | 增加短信去重校验 | v2.1 |
| 店铺名重复（旗舰店旗舰店） | 多次替换导致 | 按优先级匹配长词组优先 | v2.0 |
| 行业词汇残留（护肤品、精华） | 只替换品牌，未替换行业 | 增加行业词汇映射表 | v2.0 |
| 口语化重复（单件单件） | 口语化函数叠加 | 去重处理 | v2.0 |
| 品牌名残留 | 替换规则不全 | 完善替换映射表 | v2.0 |

### 7.2 迭代改进记录

| 版本 | 改进内容 | 日期 |
|------|----------|------|
| v1.0 | 基础替换功能 | 2026-03-30 |
| v2.0 | 增加：模板分析、行业适配、自动校验、替换标注 | 2026-03-31 |
| v2.1 | 增加：列数一致性校验、知识库条目数量校验、短信重复校验 | 2026-03-31 |

### 7.3 核心原则

1. **结构不变原则**：输出文件必须严格保持原模板的列结构，只修改内容
2. **知识库只增不减原则**：保持原有知识库条目，只修改内容或新增
3. **校验优先原则**：生成后必须通过所有校验规则

## 九、使用示例

```
用户提供：
- 活动信息：竹叶青春茶热卖季
- 模板文档：娇韵诗外呼流程模板.xlsx

处理流程：
1. 模板分析 → 识别可替换内容 + 行业词汇（护肤品）
2. 信息提取 → C1-C6 结构化
3. 内容替换 → 品牌/活动/产品 + 行业适配（护肤品→茶叶）
4. 自动校验 → R1-R8 规则检查
5. 生成输出 → 5个sheet + 标注

输出：
- 竹叶青春茶热卖季话术_校验版.xlsx
```
